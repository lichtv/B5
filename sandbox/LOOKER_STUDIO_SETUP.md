# Looker Studio Reporting Setup

Use Looker Studio as the read-only weekly reporting layer. Keep `dashboard.html` as the daily operating cockpit for lead review, prompt approval, outreach, and status updates.

## Source Sheet

Connect Looker Studio to the `AGOS_Metrics` tab generated by Apps Script.

Required fields:

- `metric_date`
- `client_id`
- `campaign_id`
- `campaign_name`
- `source`
- `valid_leads`
- `emails_sent`
- `bounces`
- `replies`
- `meetings_booked`
- `email_opens`
- `email_clicks`
- `bounce_rate`
- `reply_rate`
- `meeting_rate`
- `open_rate`
- `click_rate`
- `last_synced_at`

## Sync Metrics

After deploying `Code.js` to Apps Script, run either:

- Apps Script editor: run `syncLookerMetrics()`.
- Authenticated Web App call: `WEB_APP_URL?action=syncLookerMetrics&token=TOKEN`.

The `TOKEN` is the same token returned by the dashboard login action.

## Recommended Scorecards

- Valid Leads: sum `valid_leads`.
- Emails Sent: sum `emails_sent`.
- Bounce Rate: average `bounce_rate`.
- Reply Rate: average `reply_rate`.
- Meetings Booked: sum `meetings_booked`.

## Required Filters

- Date range: `metric_date`.
- Client: `client_id`.
- Campaign: `campaign_name`.
- Source: `source`.

## Recommended Charts

Daily send and reply trend:

- Dimension: `metric_date`
- Metrics: `emails_sent`, `reply_rate`, `meetings_booked`

Campaign comparison:

- Dimension: `campaign_name`
- Metrics: `emails_sent`, `bounce_rate`, `reply_rate`, `meetings_booked`

Lead source quality:

- Dimension: `source`
- Metrics: `valid_leads`, `reply_rate`, `meetings_booked`

Engagement signal:

- Dimension: `metric_date`
- Metrics: `email_opens`, `email_clicks`, `open_rate`, `click_rate`

## Decision Rules

- If `bounce_rate` is above `0.05`, pause sending and clean lead data.
- If `reply_rate` is below `0.02` after 200 sent emails, revise ICP, offer angle, or copy.
- If meetings are booked, review campaign/source combinations that created them.
- Treat `open_rate` as a secondary signal only.
